2  yrXlsim examples

3 Spreadsheet examples

All blocks below are yrXlsim YAML from the spec. For each example the book shows three things: the YAML source, the Formulas view of the spreadsheet (cell formulas and literals), and the Values view (evaluated results). Formulas are evaluated in the browser using HyperFormula; when the YAML includes a values override or meta.seed, those are used for reproducible Values view.

3.1 §1 Document structure

3.1.1 1.1 Minimal document

version: "0.0.2"
rows:
  - ["Label", "Data"]
  - ["X", "=A2"]

3.1.2 1.2 Minimal (rows only) — §11.1

version: "0.0.2"
rows:
  - ["first die", "second die", "Total"]
  - ["=RANDBETWEEN(1,6)", "=RANDBETWEEN(1,6)", "=A2+B2"]

3.1.3 1.3 Multi-sheet (§1.4)

When the root has a non-empty sheets array, the document has multiple sheets. Root-level rows, cells, and fill are ignored. Each sheet is processed independently.

version: "0.0.2"
sheets:
  - version: "0.0.2"
    rows:
      - ["Label", "Data"]
      - ["X", "=A2"]
  - rows:
      - ["=A1+B1", "Sum"]

3.2 §2 Rows

3.2.1 2.1 Cell value types (formula, literal, number, null, empty, literal equals)

rows:
  - ["=A2+B2", "Total", "42", null, "", "'=literal equals"]

3.2.2 2.2 Leading or middle blanks

rows:
  - ["A", "", "C"]    # B is blank
  - ["", "B2", ""]    # A and C blank
  - ["X", "Y"]        # row 3 has two cells; trailing cells are blank

3.3 §3 Cells (A1 map)

3.3.1 3.1 Sparse cells only

rows: []
cells:
  A1: "Title"
  B2: "=A1"
  C3: null

3.3.2 3.2 Sparse (cells only) — §11.2

rows: []
cells:
  A1: "Title"
  B1: "Value"
  A2: "=RANDBETWEEN(1,100)"
  B2: "=A2"

3.4 §4 Fill (expand)

3.4.1 4.1 Block fill (range + value)

rows: []
fill:
  - range: A1:C4
    value: "=RANDBETWEEN(1,6)"

3.4.2 4.2 Block fill (from/to)

fill:
  - from: A1
    to: C4
    value: "=RANDBETWEEN(1,6)"

3.4.3 4.3 Row fill down

rows:
  - ["ColA", "ColB"]
  - ["=A2+1", "=B2*2"]
fill:
  - row: 2
    down: 8

3.4.4 4.4 Row fill with toRow/toCol

rows:
  - ["X", "Y"]
fill:
  - row: 1
    toRow: 4
    toCol: D

3.4.5 4.5 Column fill right

rows:
  - ["=A1", "=B1", "=C1"]
cells:
  A1: "Header"
fill:
  - col: A
    right: 2
    down: 3

3.4.6 4.6 Cell fill down + right

cells:
  A1: "=A1+1"
fill:
  - from: A1
    down: 3
    right: 2

3.4.7 4.7 Cell fill with to

cells:
  C4: "=C4*2"
fill:
  - from: C4
    to: B2

3.4.8 4.8 Fill — row fill down (§11.5)

version: "0.0.2"
rows:
  - ["first die", "second die", "Total"]
  - ["=RANDBETWEEN(1,6)", "=RANDBETWEEN(1,6)", "=A2+B2"]
fill:
  - row: 2
    down: 8

3.4.9 4.9 Fill — cell fill down (§11.6)

version: "0.0.2"
cells:
  A1: "Score"
  A2: "=RANDBETWEEN(1,100)"
fill:
  - from: A2
    down: 5

3.4.10 4.10 Fill — block fill (§11.7)

version: "0.0.2"
rows: []
fill:
  - range: A1:C4
    value: "=RANDBETWEEN(1,6)"

3.4.11 4.11 Fill — cell fill down + right (§11.8)

version: "0.0.2"
cells:
  A1: "=RANDBETWEEN(1,6)"
fill:
  - from: A1
    down: 3
    right: 2

3.4.12 4.12 Fill — cell fill up with clamping (§13.9)

Fill from a cell up; cells that would land at row < 1 are discarded (boundary clamping).

version: "0.0.2"
cells:
  A5: "=A4*2"
fill:
  - from: A5
    up: 3

3.4.13 4.13 Fill — column fill left (§13.10)

Replicate a template column to the left; column references in formulas adjust.

version: "0.0.2"
rows:
  - [null, null, "=C1+1"]
  - [null, null, "=C2+1"]
cells:
  C1: "100"
fill:
  - col: C
    left: 2

3.5 §5 Resolution (cells override rows)

3.5.1 5.1 Resolution: cells override rows

rows:
  - ["A", "B", "C"]
  - ["1", "2", "3"]
cells:
  B2: "overridden"
  D1: "extra"

3.5.2 5.2 Hybrid (rows + cells override) — §11.3

rows:
  - ["Name", "Score"]
  - ["Alice", "=B2"]
  - ["Bob", "85"]
cells:
  B2: "=RANDBETWEEN(0,100)"
  D1: "Note"

3.5.3 5.3 Cells override fill (pipeline step 4)

After fill runs, cells is applied again. So an explicit cells entry always wins over content produced by fill.

version: "0.0.2"
rows:
  - ["Hdr", "Val"]
  - ["a", "=A2+1"]
fill:
  - row: 2
    down: 2
cells:
  B3: "overridden"

3.6 §6 Used range

3.6.1 6.1 Used range from rows and cells

rows:
  - ["X", "Y"]
  - ["a", "b", "c"]
cells:
  E1: "far"

3.7 §7 Values (reproducibility override)

The Values view below each example uses the YAML values override when present; otherwise formulas are evaluated in the browser.

3.7.1 7.1 Values — A1-keyed

rows:
  - ["=RANDBETWEEN(1,6)"]
values:
  A1: 4

3.7.2 7.2 Values — array of rows

rows:
  - ["A", "B"]
  - ["=1+1", "=A2*2"]
values:
  - [null, null]
  - [2, 4]

3.7.3 7.3 Values override (§11.4)

rows:
  - ["first die", "second die", "Total"]
  - ["=RANDBETWEEN(1,6)", "=RANDBETWEEN(1,6)", "=A2+B2"]
values:
  A2: 4
  B2: 2
  C2: 6

3.8 §8 Meta

3.8.1 8.1 Meta with suggested keys

version: "0.0.2"
rows:
  - ["Roll"]
  - ["=RANDBETWEEN(1,6)"]
meta:
  seed: 42
  cols: 5
  defaultColWidth: 12

3.9 §9 YAML-specific rules

3.9.1 9.1 Quoted key and multiline formula

rows: []
cells:
